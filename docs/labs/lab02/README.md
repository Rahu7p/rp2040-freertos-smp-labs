# Activity 1 - Independent Producer and Consumer Tasks

---

## Objective

Create two FreeRTOS tasks executing on different processor cores. Although the **Producer** continuously generates data, the **Consumer** cannot access those values because no communication mechanism has been implemented yet.

The objective of this activity is to recognize that tasks executing on different processor cores require an explicit communication mechanism to exchange information.

---

## Procedure

1. Open the **Template Project**.

2. Create two FreeRTOS tasks named:

   - `ProducerTask`
   - `ConsumerTask`

3. Configure both tasks with:

   - The same priority.
   - An infinite execution loop.
   - A delay of **500 ms**.

4. Configure task affinity so that:

   - `ProducerTask` always executes on **Core 0**.
   - `ConsumerTask` always executes on **Core 1**.

5. The `ProducerTask` should periodically generate an integer value.

Example:

```text
0
1
2
3
...
```

6. The `ConsumerTask` should periodically attempt to read a value.

Since no communication mechanism exists yet, display:

```text
???
```

7. Do **not** use any communication mechanism.

Do **not** use:

- Queue
- Binary Semaphore
- Mutex
- Event Group

8. Start the scheduler.

---

## Example Output

```text
Producer (Core 0) -> 0
Consumer value (Core 1): ???

Producer (Core 0) -> 1
Consumer value (Core 1): ???

Producer (Core 0) -> 2
Consumer value (Core 1): ???
```

Notice that the Producer continuously generates values, but the Consumer has no mechanism to receive them.

---

## Expected Behavior

After running the application:

- The Producer executes on **Core 0**.
- The Consumer executes on **Core 1**.
- The Producer continuously generates new values.
- The Consumer periodically attempts to read a value.
- The Consumer cannot access the values generated by the Producer.
- Both tasks execute independently because no communication mechanism exists between them.
