# Activity 1 - Independent Producer and Consumer Tasks

---

## Objective

Create two FreeRTOS tasks executing on different processor cores. Although the **Producer** continuously generates data, the **Consumer** cannot access those values because no communication mechanism has been implemented yet.

The objective of this activity is to recognize that tasks executing on different processor cores require an explicit communication mechanism to exchange information.

---

## Procedure

1. Open the **Template Project**.

2. Create two FreeRTOS tasks named:

   - `ProducerTask`
   - `ConsumerTask`

3. Configure both tasks with:

   - The same priority.
   - An infinite execution loop.
   - A delay of **500 ms**.

4. Configure task affinity so that:

   - `ProducerTask` always executes on **Core 0**.
   - `ConsumerTask` always executes on **Core 1**.

5. The `ProducerTask` should periodically print an incrementing integer value.

Example:

```text
Producer (Core 0) -> 0
Producer (Core 0) -> 1
Producer (Core 0) -> 2
...
```

6. The `ConsumerTask` should periodically display the following placeholder:

```text
Consumer (Core 1) <- ???
```

The question marks represent a value that the Consumer cannot obtain because no communication mechanism has been implemented yet.

7. Do **not** use any communication mechanism.

Do **not** use:

- Queue
- Binary Semaphore
- Mutex
- Event Group

8. Start the scheduler.

---

## Example Output

```text
Producer (Core 0) -> 0
Consumer (Core 1) <- ???

Producer (Core 0) -> 1
Consumer (Core 1) <- ???

Producer (Core 0) -> 2
Consumer (Core 1) <- ???
```

Notice that the Producer continuously generates values while the Consumer has no mechanism to obtain them.

---

## Expected Behavior

After running the application:

- The Producer always executes on **Core 0**.
- The Consumer always executes on **Core 1**.
- The Producer continuously generates new values.
- The Consumer periodically displays the placeholder `???`.
- The Consumer cannot obtain the values generated by the Producer.
- Both tasks execute independently because no communication mechanism exists between them.
